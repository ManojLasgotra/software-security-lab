# Experiment 06: Threat Modeling using STRIDE Framework

## Objective
To identify possible attack vectors and the attack surface of a web application through structured threat modeling.

## Methodology
1. **System Decomposition:** Created a Data Flow Diagram (DFD) for an Online Bookstore application.
2. **Threat Identification:** Applied the **STRIDE** framework to each DFD component.
3. **Risk Rating:** Ranked threats based on impact and likelihood.

## Results
* **High-Risk Threat:** Identified a potential "Elevation of Privilege" via parameter tampering in the user profile ID.
